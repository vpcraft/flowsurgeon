<task type="auto">
  <name>Task 1: Build token layer and refactor all base.html CSS</name>
  <files>src/flowsurgeon/ui/templates/base.html</files>
  <action>
This is a comprehensive single-file CSS refactor of base.html. Work through these steps IN ORDER within the single file:

**Step 1 — Remove Google Fonts.**
Delete both `<link rel="preconnect" href="https://fonts.googleapis.com">` and `<link rel="preconnect" href="https://fonts.gstatic.com">` tags, and the `<link href="https://fonts.googleapis.com/css2?family=Roboto...">` tag from `<head>`. Remove any `font-family: 'Roboto', ...` reference on `body`.

**Step 2 — Build :root {} token block at the top of the `<style>` block.**
Insert the full token system from RESEARCH.md (lines 83-164). This includes:
- Primitive color tokens (--color-gray-950 through --color-white, --color-brand)
- Method badge tokens (--color-method-get-bg/fg through --color-method-head-bg/fg)
- Status badge tokens (--color-status-2xx-bg/fg through --color-status-5xx-bg/fg)
- Duration tokens (--color-dur-fast/medium/slow)
- Semantic tokens (--bg-base, --bg-surface, --bg-elevated, --bg-sunken, --border-color)
- Text tokens (--text-primary, --text-body, --text-muted, --text-dim, --text-faint, --text-brand)
- Spacing scale (--space-1 through --space-8: 4px, 8px, 12px, 16px, 20px, 24px, 32px, 48px)
- Radii (--radius-sm: 4px, --radius-md: 8px, --radius-lg: 12px, --radius-pill: 999px)
- Font stacks (--font-sans, --font-mono per RESEARCH.md)
- Transitions (--transition-fast: 0.12s ease, --transition-base: 0.15s ease)

**Step 3 — Replace all hardcoded hex values with token references.**
Go through every CSS rule in the file. Replace every `#XXXXXX` with the appropriate `var(--token)`. Use semantic tokens (--bg-surface, --text-muted) in component rules, NOT primitive tokens (--color-gray-900). The primitive tokens exist only so semantic tokens can reference them.

Map (from RESEARCH.md color inventory):
- `#0D0F12` -> `var(--bg-base)`
- `#131720` -> `var(--bg-surface)`
- `#161A22` -> `var(--bg-elevated)`
- `#0D1117` -> `var(--bg-sunken)`
- `#1E2328` -> `var(--border-color)` (for border uses) or `var(--bg-elevated)` (for background uses -- use judgment)
- `#4B5563` -> `var(--text-faint)`
- `#6B7280` -> `var(--text-dim)`
- `#9CA3AF` -> `var(--text-muted)`
- `#E5E7EB` -> `var(--text-body)`
- `#FFFFFF` -> `var(--text-primary)`
- `#00D4AA` -> `var(--text-brand)` or `var(--color-brand)` depending on context

Also replace hardcoded px spacing values where they align with the scale (e.g., `padding: 16px` -> `padding: var(--space-4)`). Use judgment -- don't force every px value into a token if it doesn't match the scale (e.g., `1px` borders stay as `1px`).

Replace hardcoded border-radius values with radius tokens.
Replace hardcoded font-family values with font tokens.
Replace hardcoded transition values with transition tokens.

**Step 4 — Fix method badge colors to Swagger convention.**
Find the `.m-GET`, `.m-POST`, `.m-PUT`, `.m-DELETE`, `.m-PATCH` CSS rules. Update them to use the method badge tokens:
- `.m-GET { background: var(--color-method-get-bg); color: var(--color-method-get-fg); }`
- `.m-POST { background: var(--color-method-post-bg); color: var(--color-method-post-fg); }`
- `.m-PUT { background: var(--color-method-put-bg); color: var(--color-method-put-fg); }`
- `.m-DELETE { background: var(--color-method-delete-bg); color: var(--color-method-delete-fg); }`
- `.m-PATCH { background: var(--color-method-patch-bg); color: var(--color-method-patch-fg); }`
- Add `.m-HEAD { background: var(--color-method-head-bg); color: var(--color-method-head-fg); }` if not present.

Ensure badges use pill shape: `border-radius: var(--radius-pill)`, `padding: 3px 10px`, `font-weight: 700`, `font-size: 11px` (or similar small size). White text on colored background -- but actually per the token design, the fg color IS the bright color and bg is the dark tint, so it's bright-on-dark. This is correct for a dark theme.

**Step 5 — Replace box-shadows with borders.**
Find ALL `box-shadow` rules in the file. Remove them. Ensure the affected selectors (`.card`, `.table-wrap`, `.panel-card`, `.stat-badge`, and any others) have `border: 1px solid var(--border-color)` instead. If they already have a border, just remove the shadow.

**Step 6 — Establish label/value typographic hierarchy.**
Define `.label` and `.value` / `.value-prominent` classes per RESEARCH.md:
```css
.label { font-size: 11px; font-weight: 600; color: var(--text-dim); letter-spacing: 0.08em; text-transform: uppercase; }
.value { color: var(--text-body); font-weight: 500; }
.value-prominent { color: var(--text-primary); font-weight: 700; }
```
Update existing `.card-label` / `.card-value` to use these tokens if they don't already. Ensure consistency.

**Step 7 — Define utility classes.**
Add the full set of utility classes from RESEARCH.md (lines 237-271) to the end of the `<style>` block:
- `.u-mono`, `.u-text-xs` through `.u-text-lg`, `.u-muted` through `.u-bright`
- `.u-flex`, `.u-flex-col`, `.u-items-center`, `.u-gap-1` through `.u-gap-4`, `.u-flex-1`, `.u-shrink-0`, `.u-min-h-0`
- `.u-scroll-y`, `.u-hidden`

Also add `code { font-family: var(--font-mono); }` reset per RESEARCH.md pitfall 2.

**Step 8 — Update status badge colors.**
Find status code badge rules (e.g., `.s-2xx`, `.s-3xx` or however they're currently named). Update to use the status tokens:
- 2xx: `background: var(--color-status-2xx-bg); color: var(--color-status-2xx-fg);`
- 3xx/4xx: `background: var(--color-status-3xx-bg); color: var(--color-status-3xx-fg);`
- 5xx: `background: var(--color-status-5xx-bg); color: var(--color-status-5xx-fg);`

Also update duration-based color rules if they exist, using --color-dur-fast/medium/slow.

**IMPORTANT:** Do NOT change any CSS class names that are generated by Python code in `panel.py` (like `.m-GET`, status classes). Only change the CSS property values inside the rules. Check `panel.py` for the `_status_class` filter to confirm class name format before changing anything.

**IMPORTANT:** Do NOT touch `panel.css` -- it stays independent per locked decision.
