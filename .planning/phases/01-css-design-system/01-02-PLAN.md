  <action>
For each template file, find every `style="..."` attribute and convert it to the appropriate utility class(es). Work file by file:

**home.html (~5 inline styles):**
Common patterns to replace:
- `style="font-family: monospace"` or `style="font-family:monospace"` -> `class="u-mono"` (append to existing class if present)
- `style="color: #9CA3AF"` or similar muted colors -> `class="u-muted"` or `class="u-dim"`
- `style="display: none"` -> `class="u-hidden"` (but check if Alpine.js x-show controls it -- if so, may need x-cloak instead)
- `style="width: Xpx"` on inputs/selects -> add a specific class in base.html if no utility fits, OR use a reasonable utility. If the width is a fixed layout concern, it may need a small component class added to base.html's style block.

**detail.html (~5 inline styles):**
- `style="margin-top: ..."` -> use spacing token class or add to component
- `style="font-family: monospace"` -> `class="u-mono"`
- `style="color: ..."` -> appropriate color utility
- `style="max-height: ..."` -> add a utility if needed (e.g., `.u-max-h-md { max-height: 400px; }`)

**detail_profile.html (~35 inline styles -- LARGEST workload):**
This partial is almost entirely inline-styled. Common patterns:
- `style="display:flex; ..."` -> `class="u-flex u-items-center u-gap-2"` etc.
- `style="font-family:monospace; font-size:12px"` -> `class="u-mono u-text-sm"`
- `style="color:#9CA3AF"` -> `class="u-muted"`
- `style="color:#6B7280"` -> `class="u-dim"`
- `style="flex:1"` -> `class="u-flex-1"`
- `style="flex-shrink:0"` -> `class="u-shrink-0"`
- `style="overflow-y:auto"` -> `class="u-scroll-y"`
- `style="min-height:0"` -> `class="u-min-h-0"`
- **EXCEPTION:** `style="width:{{ pct }}%; ..."` on line ~63 -- this is a Jinja2-interpolated dynamic value. It MUST remain as inline style. Only convert the static parts of that style attribute if possible (e.g., if it also has `background-color`, convert that part to a class and leave only `style="width:{{ pct }}%"`).

**detail_sql.html (~10 inline styles):**
- Same pattern: flex layouts, padding, colors, font-family -> utility classes

**detail_traceback.html (~10 inline styles):**
- Same pattern: flex layouts, colors, font-family -> utility classes

**General rules:**
1. When an element already has a `class=` attribute, APPEND the utility classes (e.g., `class="existing-class u-mono u-text-sm"`).
2. When an element has ONLY a `style=` attribute and no class, replace with `class="..."`.
3. If a style property has no matching utility class AND it's used in multiple places, add a new utility class to base.html's `<style>` block. Name it with the `u-` prefix. Keep additions minimal.
4. If a style property is unique to one element and trivial, consider adding it as a component-level class rather than a utility.
5. After all conversions, do a final grep to confirm no static `style=` remains (only dynamic Jinja2 ones).
