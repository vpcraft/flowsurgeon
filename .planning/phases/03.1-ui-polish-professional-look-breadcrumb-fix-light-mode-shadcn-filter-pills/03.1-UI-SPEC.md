# Phase 03.1 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — pure CSS custom properties in `base.html` `<style>` block |
| Preset | not applicable |
| Component library | none (Alpine.js for interactivity only) |
| Icon library | none |
| Font | System stack: `-apple-system, BlinkMacSystemFont, 'Inter', 'Segoe UI', sans-serif` (monospace: `'JetBrains Mono', 'Fira Code', 'Cascadia Code', ui-monospace, monospace`) |

## Spacing Scale

Declared values from existing `:root {}` tokens — light mode adds no new spacing tokens:

| Token | Value | Usage |
|-------|-------|-------|
| `--space-1` | 4px | Icon gaps, inline padding |
| `--space-2` | 8px | Compact element spacing |
| `--space-4` | 16px | Default element spacing (content area vertical padding) |
| `--space-6` | 24px | Section padding (content area horizontal padding) |
| `--space-7` | 32px | Layout gaps (REDUCE: content area padding currently uses this — reduce to `--space-4`) |
| `--space-8` | 48px | Major section breaks (REDUCE: content area padding currently uses this — reduce to `--space-6`) |

`--space-3` (12px) and `--space-5` (20px) are excluded — not on the standard 8-point scale (4, 8, 16, 24, 32, 48, 64).

**Phase changes to spacing:**

- `.content` padding: change from `var(--space-7) var(--space-8)` (32px/48px) to `var(--space-4) var(--space-6)` (16px/24px)
- `.tr` row height: change from 52px to 40px (multiple of 4; within 36–40px target)
- `.tr-req` row height: change from 48px to 40px (unified with `.tr`)
- Topbar height: retain at 64px (Swagger reference confirms this is standard)
- Toggle group internal padding: 4px vertical, `--space-4` horizontal per item

Exceptions: Topbar height 64px is intentionally NOT 8-point-grid-derived; matches Swagger UI reference.

## Typography

Existing type scale retained — no new sizes introduced. Phase does not change typography.

| Role | Size | Weight | Line Height | Token class |
|------|------|--------|-------------|-------------|
| Label / column header | 11px | 600 (semibold) | 1.4 | `.label`, `.th`, `.card-label` |
| Body / table cell | 13px | 400 (regular) | 1.5 | `.td`, `.field-val` |
| Base / breadcrumb / filter | 14px | 400 | 1.5 | `.breadcrumb`, `.empty` |
| Heading / card value | 24–28px | 600 (semibold) | 1.2 | `.section-title`, `.card-value` |

Exactly 2 weights in use: 400 (regular) for body/data/breadcrumb, 600 (semibold) for all labels, headings, active states, and emphasis. Weight 500 and 700 are not used — consolidated into 600. `.b-mid` breadcrumb middle segment uses `font-weight: 600`.

|------|-------|-------|
| Dominant (60%) | `#0D0F12` (`--bg-base`) | Page background |
| Secondary (30%) | `#131720` / `#0D1117` (`--bg-surface` / `--bg-sunken`) | Table container, topbar, panel cards |

## Component Contracts

### 1. Breadcrumb Fix (detail.html)

**Current state (broken):** `<span class="logo">FlowSurgeon</span>` + `<a class="b-dim">FlowSurgeon</a>` — duplicated "FlowSurgeon" text, logo is not a link.

**Target structure — detail page (request detail):**
```
[FlowSurgeon logo link]  /  [GET badge] /books  /  abc12345
```

Markup pattern:
```html
<nav class="topbar">
  <a class="logo" href="{{ debug_route }}">FlowSurgeon</a>
  <div class="breadcrumb">
    <span class="sep">/</span>
    <a class="b-mid" href="{{ debug_route }}?method={{ record.method }}&path={{ record.path | urlencode }}">
      <span class="method {{ record.method | method_class }}">{{ record.method }}</span>
      <span class="u-mono u-text-sm">{{ record.path }}</span>
    </a>
    <span class="sep">/</span>
    <span class="b-cur">{{ record.request_id[:8] }}</span>
  </div>
</nav>
```

Separator: forward slash `/` (as text, not `&rsaquo;`). Logo is `<a class="logo">` — inherits existing `.logo` font-weight and brand color, gains `href`.

**Target structure — home page:**
```html
<nav class="topbar">
  <a class="logo" href="{{ debug_route }}">FlowSurgeon</a>
</nav>
```

### 2. Toggle Group (replaces filter pills — home.html)

Replace `.filter-pills` / `.pill` / `.pill-active` pattern with a shadcn-style toggle group: single bordered container, segmented buttons inside.

**Visual spec:**
- Container: `background: var(--bg-surface); border: 1px solid var(--border-color); border-radius: var(--radius-md); display: inline-flex; padding: 4px;`
- Inactive item: `background: transparent; color: var(--text-dim); padding: 4px var(--space-4); font-size: 13px; font-weight: 400; border-radius: var(--radius-sm); border: none; cursor: pointer; transition: var(--transition-fast);`
- Inactive hover: `background: var(--bg-elevated); color: var(--text-body);`
- Active item: `background: var(--bg-elevated); color: var(--text-primary); font-weight: 600;` — NOT brand green
- Active item: NO brand color — surface fill + white/bold text only

**Layout — controls row:**
```
[Toggle group: ALL | GET | POST | PUT | DELETE | PATCH]    [12 routes]
```
Route count is right-aligned on the same flex row as the toggle group. Use `justify-content: space-between` on the parent row.

**CSS classes to add:**
```css
.toggle-group { ... }
.toggle-item { ... }
.toggle-item:hover { ... }
.toggle-item-active { ... }
```

**CSS classes to REMOVE:** `.filter-pills`, `.pill`, `.pill-active`

Position: content area, above routes list (current `.filter-pills` position — keep location, replace markup and styles).

### 3. Animation Removal (base.html)

**Remove entirely:**
- `@keyframes fadeIn { ... }` block (lines 108–111)
- `@keyframes cardIn { ... }` block (lines 113–116)
- `@keyframes slideDown { ... }` block (lines 118–121)
- `animation: slideDown 0.2s ease both;` from `.topbar`
- `animation: slideDown 0.2s ease 0.05s both;` from `.subnav`
- `animation: fadeIn 0.2s ease both;` from `.content`
- `animation: cardIn 0.2s ease both;` from `.req-card`
- All staggered card animation-delay rules (`.req-card:nth-child(1)` through `:nth-child(12)`)

**Keep:** All `transition:` properties — hover/focus transitions on interactive elements stay.

Also remove from `.req-card`: `transform: translateY(-2px)` on hover (keep `border-color` change only). The translateY lift is an animation effect.

### 4. Spacing Density Tightening (base.html)

| Selector | Property | Old Value | New Value |
|----------|----------|-----------|-----------|
| `.content` | `padding` | `var(--space-7) var(--space-8)` (32px/48px) | `var(--space-4) var(--space-6)` (16px/24px) |
| `.tr` | `height` | `52px` | `40px` |
| `.tr-req` | `height` | `48px` | `40px` |

These are the only density changes. Topbar (64px), subnav (48px), and card/panel padding remain unchanged.
