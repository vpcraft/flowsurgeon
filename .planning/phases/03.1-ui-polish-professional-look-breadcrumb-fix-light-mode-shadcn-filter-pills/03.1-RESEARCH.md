### Pattern 1: CSS Custom Property Light Mode Override

**What:** Add a single `@media (prefers-color-scheme: light)` block after the `:root {}` block in `base.html`. Reassign only semantic tokens; leave primitive and method badge tokens unchanged.

**When to use:** Always use this pattern for system-preference light mode when no JS toggle is needed.

**Example:**
```css
/* After existing :root {} block in base.html */
@media (prefers-color-scheme: light) {
  :root {
    --bg-base:      #FFFFFF;
    --bg-surface:   #F8F9FA;
    --bg-elevated:  #F1F3F5;
    --bg-sunken:    #EBEDF0;
    --border-color: #DEE2E6;

    --text-primary: #0D0F12;
    --text-body:    #1C1E21;
    --text-muted:   #495057;
    --text-dim:     #6C757D;
    --text-faint:   #ADB5BD;
    --text-brand:   #00A884;

    /* Status badge adjustments for light bg */
    --color-status-2xx-bg: #D1FAE5;
    --color-status-2xx-fg: #065F46;
    --color-status-3xx-bg: #FEF3C7;
    --color-status-3xx-fg: #92400E;
    --color-status-4xx-bg: #FEF3C7;
    --color-status-4xx-fg: #92400E;
    --color-status-5xx-bg: #FEE2E2;
    --color-status-5xx-fg: #991B1B;

    /* Duration adjustments for light bg */
    --color-dur-fast:   #059669;
    --color-dur-medium: #D97706;
    --color-dur-slow:   #DC2626;
  }

  /* Hardcoded scrollbar thumb — must override explicitly */
  ::-webkit-scrollbar-thumb {
    background: #CED4DA;
  }
}
```

Note: Three values in `base.html` are hardcoded outside `:root` and require separate overrides:
1. `#313244` (scrollbar thumb) — override in `@media` block targeting `::-webkit-scrollbar-thumb`
2. `#C8D6E5` (`.cs-2xx` text color) — this is a card-view status color; acceptable to leave as-is since it already works on both backgrounds
3. `#00BFA0` (`.filter-btn:hover`) — filter button is not in the toggle group; acceptable to leave

**What:** Single bordered container `div.toggle-group` with `button.toggle-item` children. Alpine.js `:class` binding on each item sets `.toggle-item-active`. No external library.

**When to use:** When project has no build step and needs a segmented control UI.

**Example:**

  gap: 2px;

  border: none;
  border-radius: var(--radius-sm);

  font-weight: 400;
  color: var(--text-dim);

  transition: background var(--transition-fast), color var(--transition-fast);

### Anti-Patterns to Avoid

- **Changing method badge token values:** `--color-method-*-bg/fg` are identical in light and dark mode by design. Do not add them to the light mode override block.
- **Using brand green for toggle active state:** Active toggle item uses `--bg-elevated` surface fill + bold white/dark text. Brand green is reserved for nav underlines, focus rings, and current breadcrumb text.
- **Adding a `[data-theme]` attribute system:** No manual toggle exists; `prefers-color-scheme` is the only trigger.
- **Changing `.subnav` animation without removing it:** The `animation: slideDown` on `.subnav` must be removed entirely (not just reduced), same as on `.topbar` and `.content`.
- **Forgetting the hardcoded scrollbar thumb:** `#313244` at line ~1076 is outside `:root` and must be overridden explicitly in the `@media` block.

## Common Pitfalls

### Pitfall 1: Leaving the `controls-row` Route Count Outside the New Flex Row

**What goes wrong:** The route count `<div>` is currently on a separate `<div>` below the filter pills (lines 26–29 in `home.html`). If it stays separate instead of being moved into the same flex row as the toggle group, it renders below instead of right-aligned on the same line.

**Why it happens:** Easy to only replace the `.filter-pills` div and forget the sibling route count div.

**How to avoid:** The toggle group and route count must be children of the same `.controls-row` flex container with `justify-content: space-between`. Remove the separate route count `<div>` entirely.

**Warning signs:** Route count appears below the toggle group on a separate line.

### Pitfall 2: `.breadcrumb` Class Has a `.sep` Naming Conflict

**What goes wrong:** `base.html` defines two `.sep` rules:
1. `.breadcrumb .sep` — color `var(--text-faint)` (line 203)
2. `.sep` standalone — `height: 1px; background: var(--border-color)` (line 373)

The breadcrumb separator `/` uses `.sep` inside `.breadcrumb`, so it correctly gets the breadcrumb-specific style. This is working correctly in the existing code. Do not consolidate or rename these classes.

**Warning signs:** Breadcrumb separators render as 1px horizontal rules instead of text.

### Pitfall 3: Alpine.js `x-data` Scope and Toggle Group Placement

**What goes wrong:** If the toggle group markup is placed outside the `x-data="{ methodFilter: 'ALL' }"` wrapper div in `home.html`, the `:class` binding will fail silently (no active state).

**Why it happens:** The `x-data` wrapper is the `<div class="spa-wrapper" x-data="{ methodFilter: 'ALL' }">` on line 12 of `home.html`. The current `.filter-pills` block is inside this wrapper. The replacement toggle group must also be inside this wrapper.

**How to avoid:** Keep the toggle group as a child of `.spa-wrapper`, replacing the `.filter-pills` div at the same DOM level.

**Warning signs:** Clicking toggle group items has no effect; no active class applied.

### Pitfall 4: Missing `font-family` on Toggle Group Buttons

**What goes wrong:** `<button>` elements do not inherit `font-family` by default in all browsers. The existing `.pill` class has `font-size: 0.8125rem` but no explicit `font-family`. The toggle group may render in the browser's default button font (serif or system-ui variant) instead of the sans-serif stack.

**How to avoid:** Include `font-family: var(--font-sans);` in `.toggle-item` CSS rule.

**Warning signs:** Toggle buttons render in a different font from surrounding text.

### Pitfall 5: `&rsaquo;` vs `/` Breadcrumb Separator in detail.html

**What goes wrong:** The current breadcrumb in `detail.html` uses `&rsaquo;` (›) as separator. The locked decision requires `/`. There are two `.sep` spans in the breadcrumb — both must be changed from the HTML entity to literal `/` text.

**How to avoid:** Replace both `<span class="sep">&rsaquo;</span>` instances with `<span class="sep">/</span>`.

### Pitfall 6: `req-card` Has Both `animation:` and Staggered Delay Rules

**What goes wrong:** Removing the `animation: cardIn 0.2s ease both;` line from `.req-card` but leaving the `.req-card:nth-child(N)` `animation-delay` rules results in harmless-but-noisy dead CSS. Both must be removed.

**How to avoid:** Search for all occurrences of `animation-delay` in `base.html` and remove the entire stagger block (`.req-card:nth-child(1)` through `:nth-child(12)`).
