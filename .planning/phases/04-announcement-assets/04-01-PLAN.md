1. In `pyproject.toml`, add a new dependency group after the `examples` group:
```toml
screenshots = ["playwright>=1.40"]
```

2. Create `scripts/capture_assets.py` — a single Python script that:

**Server lifecycle:**
- Starts the FastAPI demo with `subprocess.Popen(["uv", "run", "--group", "examples", "uvicorn", "examples.fastapi.demo_fastapi:app", "--host", "127.0.0.1", "--port", "8765"])`
- Uses `try/finally` to guarantee `server.terminate()` and `server.wait()` cleanup
- Waits 3 seconds after startup for server readiness

**Data seeding (before any screenshots):**
- Uses `page.goto()` to hit each demo route to populate the database with realistic data: `/books`, `/books/1`, `/books/duplicates`, `/books/slow`, `/slow`
- Skips `/boom` to avoid a 500 error contaminating the data seed (or hits it last — it's fine for screenshot data since it shows a 500 status)
- After seeding, waits 1 second for data to settle

**Playwright setup:**
- Uses `sync_playwright()` context manager
- Launches Chromium headless
- Creates page with `viewport={"width": 1280, "height": 800}` and `color_scheme="dark"` (CRITICAL: forces dark mode since Phase 3.1 added light mode via @media prefers-color-scheme)

**Screenshot 1 — Routes home (`docs/screenshots/routes-home.png`):**
- Navigate to `http://127.0.0.1:8765/flowsurgeon`
- `page.wait_for_load_state("networkidle")`
- `page.wait_for_selector(".tr", timeout=5000)` — waits for Alpine.js to render route rows
- `page.wait_for_timeout(500)` — extra settle time for Alpine x-show directives
- `page.screenshot(path="docs/screenshots/routes-home.png", full_page=False)`

**Screenshot 2 — Request detail SQL tab (`docs/screenshots/request-detail-sql.png`):**
- Navigate to `http://127.0.0.1:8765/flowsurgeon` (routes home)
- Wait for route rows to render (same wait pattern as above)
- Click the route row for `/books/slow` which has slow+dup SQL badges. Use a locator that finds the `.tr` element containing the text `/books/slow`: `page.locator("a.tr", has_text="/books/slow").first.click()`
- This navigates to the route_detail page (filtered request list for that route)
- Wait for load, then click the first request row: `page.locator("a.tr").first.click()`
- This navigates to the detail page. Wait for load.
- Click the SQL tab: `page.locator(".subnav-item", has_text="SQL").click()`
- `page.wait_for_timeout(500)` — wait for Alpine tab switch
- `page.screenshot(path="docs/screenshots/request-detail-sql.png", full_page=False)`

**GIF frame capture (`docs/demo/frames/frame_NNNN.png`):**
- Create `docs/demo/frames/` directory
- Use a global `frame_num` counter and a helper function `capture_frames(page, frames_dir, count)` that captures `count` identical frames (to create a "hold" effect at that fps)
- Target 10 fps, so 10 frames = 1 second hold

GIF walkthrough sequence:
  1. Routes home page — hold 2 seconds (20 frames): Navigate to `/flowsurgeon`, wait for render, capture 20 frames
  2. Click `/books/slow` route row — hold 1.5 seconds (15 frames): Click, wait for route_detail page load, capture 15 frames
  3. Click first request in the list — hold 1.5 seconds (15 frames): Click first `.tr`, wait for detail page, capture 15 frames
  4. Click SQL tab — hold 2.5 seconds (25 frames): Click `.subnav-item` with text "SQL", wait for tab switch, capture 25 frames

Total: ~75 frames at 10fps = ~7.5 seconds. This is intentionally shorter than the 20-30s target because a tight demo is more compelling.

**GIF assembly (ffmpeg two-pass palette):**
- After frame capture, run ffmpeg subprocess for two-pass palette GIF:
  - Pass 1: `ffmpeg -y -framerate 10 -i docs/demo/frames/frame_%04d.png -vf "scale=1280:-1:flags=lanczos,palettegen=max_colors=256:stats_mode=diff" docs/demo/palette.png`
  - Pass 2: `ffmpeg -y -framerate 10 -i docs/demo/frames/frame_%04d.png -i docs/demo/palette.png -lavfi "fps=10,scale=1280:-1:flags=lanczos [x]; [x][1:v] paletteuse=dither=bayer:bayer_scale=3:diff_mode=rectangle" docs/demo/demo.gif`
- After GIF is produced, optionally clean up `docs/demo/frames/` directory and `docs/demo/palette.png` (delete them to keep repo clean)

**Script should be runnable with:**
```bash
uv run --group screenshots python scripts/capture_assets.py
```

3. Create necessary directories before running: `docs/screenshots/`, `docs/demo/`

4. Run `uv sync --group screenshots` then `uv run --group screenshots playwright install chromium` to install dependencies.

5. Run the capture script: `uv run --group screenshots python scripts/capture_assets.py`
