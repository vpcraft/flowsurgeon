**panel.py** -- Add `render_route_detail_page()` after `render_routes_page()`:

```python
def render_route_detail_page(
    records: list[RequestRecord],
    debug_route: str,
    route_method: str,
    route_path: str,
    status: str = "",
    sort: str = "recent",
    page: int = 1,
    show: int = 25,
) -> str:
    """Return the filtered request list for a specific route."""
    filtered = _filter_records(records, method=route_method, path=route_path, status=status)

    # Sort
    if sort == "duration":
        filtered = sorted(filtered, key=lambda r: r.duration_ms, reverse=True)
    elif sort == "status":
        filtered = sorted(filtered, key=lambda r: r.status_code)
    else:  # "recent" (default) -- most recent first
        filtered = sorted(filtered, key=lambda r: r.timestamp, reverse=True)

    page_items, total_pages, page = _paginate(filtered, page, show)
    tmpl = _env.get_template("route_detail.html")
    return tmpl.render(
        records=page_items,
        total_records=len(filtered),
        route_method=route_method,
        route_path=route_path,
        debug_route=debug_route,
        status=status,
        sort=sort,
        page=page,
        total_pages=total_pages,
        show=show,
    )
```

**route_detail.html** -- New template. Extends `base.html`.

Structure:
1. Topbar with 2-level breadcrumb:
   - `<a href="{{ debug_route }}">FlowSurgeon</a> > <span class="method {{ route_method | method_class }}">{{ route_method }}</span> <span class="u-mono u-text-sm">{{ route_path }}</span>`

2. Filter/sort controls bar:
   - Status filter: `<select>` with options: All, 2xx, 3xx, 4xx, 5xx. Current value from `status` context var.
   - Sort: `<select>` with options: Recent, Duration, Status. Current value from `sort` context var.
   - Show per page: `<select>` with options: 10, 25, 50. Current value from `show` context var.
   - Controls submit as query params appended to `{{ debug_route }}?method={{ route_method }}&path={{ route_path | urlencode }}&status=X&sort=Y&show=Z`
   - Use standard `<form method="GET">` with hidden inputs for method and path, so the filter controls work as a single form submission.

3. Results summary: `{{ total_records }} requests for [METHOD] {{ route_path }}`

4. Request list table (reuse `.tr` pattern from home.html):
   - Header row (optional): Request ID, Status, Duration, Queries, Time
   - For each request in `records`:
     - Row links to `{{ debug_route }}/{{ record.request_id }}`
     - Columns: request ID (truncated to 8 chars), status badge, duration with duration_class, query count, relative time
   - Empty state: "No requests recorded for this route"

5. Pagination controls:
   - Previous/Next links
   - Page indicator: `Page {{ page }} of {{ total_pages }}`
   - Links preserve all current query params (method, path, status, sort, show) and change only `page`

Use Jinja2's `urlencode` filter for the path param in all URLs. This handles curly braces in paths like `/api/users/{id}` (Pitfall 5).
